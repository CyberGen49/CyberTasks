CyberTasks

API Documentation

What's it about?

The CyberTasks API works just like any run-of-the-mill REST API, meaning that you can interact with it using the HTTP requesting features of your favourite programming language. Each API endpoint only works with its listed method (be it GET, POST, PUT, etc.), so be sure you're using the correct methods in your code.

Authentication

Most of the CyberTasks API requires authentication, which can be accomplished by using an API key. The endpoints that require authentication are marked with an badge below.

Creating an API key

You can create an API key by signing into CyberTasks, then navigating to settings and locating the API keys section. From here, create a new key by clicking the Create key button, or modify the attributes of existing keys.

For security reasons, you'll select a list of scopes for each API key, corresponding to the API endpoints that you want to use. You can see each endpoint's scope below.

Using your API key to authenticate requests

Authenticating API requests is accomplished through the use of custom HTTP headers, so set the CyberTasks-Key header to your API key in your requests to authenticate.

Other notes

All endpoints with methods other than expect the request body to contain valid JSON with values for that endpoint's JSON body parameters. Additionally, these requests are expected to have a Content-Type of application/json.
