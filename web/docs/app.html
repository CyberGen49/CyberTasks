CyberTasks

API Documentation

What's it about?

The CyberTasks API works just like any run-of-the-mill REST API, meaning that you can interact with it using the HTTP requesting features of your favourite programming language. Each API endpoint only works with its listed method, so be sure you're using the correct methods in your code.

Semantic notes

As a general rule of thumb for all REST APIs:
requests are for getting resources,
requests are for creating resources,
requests are for editing resources, and
requests are for deleting resources.

Path parameters refer to segments of the API endpoint path that contain dynamic data, like IDs. These are indicated by less than (<) and greater than (>) signs, which shouldn't be included as part of your final request.

Query parameters refer to key-value pairs placed at the end of the URL, after a question mark (?), separated by ampersands (&). Example: /path/to/endpoint?a=foo&b=bar

JSON body parameters refer to parameters that are passed as a JSON string contained within the request body. A JSON body is required for all and requests. Additionally, these requests are expected to have a Content-Type header of application/json.

The API responds to requests with a JSON body containing status information along with the requested data. This response body is intended to be parsed and accessed as an object in your programing language.

Authentication

Most of the CyberTasks API requires authentication, which can be accomplished by using an API key. The endpoints that require authentication are marked with an badge below.

Creating an API key

You can create an API key by signing into CyberTasks, then navigating to settings and locating the API keys section. From here, create a new key by clicking the Create key button, or modify the attributes of existing keys.

For security reasons, you'll select a list of scopes for each API key, corresponding to the API endpoints that you want to use. You can see each endpoint's scope below.

Using your API key to authenticate requests

Authenticating API requests is accomplished through the use of custom HTTP headers. Set the CyberTasks-Key header of your request to your API key to authenticate.

Authenticating with a user access token (not recommended)

It's also possible to gain unrestricted access to the API (including undocumented endpoints) by using a user access token for authentication. This token can be acquired by right-clicking or long-pressing on your profile picture within CyberTasks, and choosing the Copy access token... option.

Note that these tokens are login-specific, are deleted when you log out, and expire if left unused for 30 days. They're also able to perform potentially dangerous actions like deleting your account, so use a normal API key if at all possible.

To authenticate with a user token, use the same custom header method as above, but with the CyberTasks-Token header instead.
